# Refactorización del Frontend de Autenticación: Login y Register Separados

## Contexto del Proyecto
Se está reestructurando la interfaz de autenticación en la carpeta `motor_antifraude/front/auth/`. Anteriormente todo estaba en un solo archivo, pero ahora se dividirá en dos vistas independientes:
1. `login.html` y su script `login.js`
2. `register.html` y su script `register.js`

La aplicación está dockerizada. El backend es un servicio en FastAPI que corre en el puerto 8000 (`http://localhost:8000`).

## REGLA ESTRICTA Y ABSOLUTA
**PROHIBIDO MODIFICAR EL BACKEND:** Debes analizar el código del backend (FastAPI) para entender los endpoints, pero **NO DEBES** alterar, modificar ni sugerir cambios en ningún archivo `.py`. Tu único objetivo es crear los archivos frontend (`.html` y `.js`) para que se conecten correctamente a la API existente.

## Tareas a realizar

### 1. Limpieza Total de Biometría
- En los nuevos archivos generados, **NO** debe existir ningún código HTML o JS relacionado con la solicitud de la foto de la cédula, etiquetas `<video>`, `<canvas>`, o captura de cámara.

### 2. Flujo de Inicio de Sesión (`login.html` y `login.js`)
- **Interfaz (`login.html`):** - Crea un formulario limpio usando Tailwind CSS que solicite `email` y `password`.
  - Agrega un enlace claro en la parte inferior que diga "¿No tienes cuenta? Regístrate aquí" que redirija a `register.html`.
- **Lógica (`login.js`):**
  - Captura el evento `submit` del formulario.
  - Realiza una petición `POST` a `http://localhost:8000/v1/auth/login`.
  - **Formato:** Envía los datos como `application/json`.
    ```json
    {
      "email": "user@example.com",
      "password": "string"
    }
    ```
  - **Manejo de respuesta:** Si es exitoso (retorna token), guárdalo en `localStorage` y redirige a la página principal de la app (ej. `../index.html`). Si falla, muestra una alerta de error.

### 3. Flujo de Registro (`register.html` y `register.js`)
- **Interfaz (`register.html`):**
  - Crea un formulario con Tailwind CSS que solicite los siguientes campos obligatorios:
    - Correo electrónico (`email`)
    - Nombre de usuario (`username` - min 3, max 50 chars)
    - Contraseña (`password` - min 8 chars)
    - Cédula (`cedula` - min 6, max 20 chars)
  - Agrega un enlace en la parte inferior que diga "¿Ya tienes cuenta? Inicia sesión" que redirija a `login.html`.
- **Lógica (`register.js`):**
  - Captura el evento `submit` del formulario.
  - Realiza una petición `POST` a `http://localhost:8000/v1/auth/register`.
  - **Formato:** La petición DEBE enviarse como `multipart/form-data` usando `FormData()` de JavaScript.
  - Añade `email`, `username`, `password` y `cedula` al `FormData`.
  - El backend espera un campo opcional `face_photo`. Envíalo vacío o gestiona el FormData de manera que el backend de FastAPI no falle por la falta del archivo (solo envía los campos de texto).
  - **Manejo de respuesta:** Si el registro es exitoso, muestra un mensaje de éxito y redirige automáticamente a `login.html`. Si hay error de validación, muestra las alertas correspondientes.
